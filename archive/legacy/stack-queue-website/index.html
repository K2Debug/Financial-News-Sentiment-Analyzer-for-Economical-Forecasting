Stack

A stack is like a pile of plates — you always add and remove from the top. The last item placed on the stack is the first one removed.

Key Operations

  • push(x) Add element x to the top
  • pop() Remove and return the top element
  • peek() View the top element without removing it
  • isEmpty() Check if the stack has no elements

Real-World Examples

  • Undo / Redo — each action is pushed; undo pops the last action
  • Browser back button — visited pages stored in a stack
  • Function call stack — nested function calls push/pop frames
  • Expression evaluation — parsing math with parentheses

View Python implementation 
class Stack:
    def __init__(self):
        self.items = []

    def push(self, item):
        self.items.append(item)   # O(1)

    def pop(self):
        if self.is_empty():
            raise IndexError("pop from empty stack")
        return self.items.pop()   # O(1)

    def peek(self):
        if self.is_empty():
            raise IndexError("peek from empty stack")
        return self.items[-1]

    def is_empty(self):
        return len(self.items) == 0

Queue

A queue is like a line at a store — the first person in line is the first one served. Items enter at the rear and leave from the front.

Key Operations

  • enqueue(x) Add element x to the rear
  • dequeue() Remove and return the front element
  • front() View the front element without removing it
  • isEmpty() Check if the queue has no elements

Real-World Examples

  • Print queue — documents printed in the order received
  • Task scheduling — CPU processes jobs first-come-first-served
  • Breadth-first search (BFS) — explore graph level by level
  • Customer service lines — tickets handled in arrival order

View Python implementation 
from collections import deque

class Queue:
    def __init__(self):
        self.items = deque()

    def enqueue(self, item):
        self.items.append(item)       # O(1)

    def dequeue(self):
        if self.is_empty():
            raise IndexError("dequeue from empty queue")
        return self.items.popleft()   # O(1)

    def front(self):
        if self.is_empty():
            raise IndexError("front from empty queue")
        return self.items[0]

    def is_empty(self):
        return len(self.items) == 0

Stack vs Queue

Both store collections of items, but they differ in how items are added and removed.

Property Stack Queue
Ordering principle LIFO — Last In, First Out FIFO — First In, First Out
Add operation push() — top enqueue() — rear
Remove operation pop() — top dequeue() — front
Analogy Stack of plates Line at a ticket counter
Typical use Undo, backtracking, recursion Scheduling, BFS, buffering
Access point One end only (top) Two ends (front & rear)
